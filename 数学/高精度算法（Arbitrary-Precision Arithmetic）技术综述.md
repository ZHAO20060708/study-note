## 1. 算法定义与必要性

在计算机标准指令集中，算术逻辑单元（ALU）通常仅支持固定位宽（如 32 位、64 位）的整数运算。当数值规模超出原生数据类型（如 C++ `uint64_t`）的表示上限（$2^{64}-1$）时，硬件层面的算术运算将发生溢出。高精度算法通过软件模拟的方式，实现对任意长度数值的精确运算。

## 2. 数据结构实现

### 2.1 存储模型

- **线性容器**：采用动态数组（如 C++ `std::vector`）或字符串存储。
    
- **基数选择**：
    
    - **位权模型**：通常以 $10^1$ 为基数（每个单元存 1 位），便于输入输出转换。
        
    - **压位优化**：为提高时空效率，可采用 $10^9$ 或 $2^{32}$ 作为基数（压位存储），充分利用 64 位整型的中间计算空间。
        
- **端序（Endianness）**：强制采用**小端序存储**（Little-endian），即数组索引 `0` 存储数值的最低位（个位）。此举可确保进位操作在线性时间复杂度下完成，无需移动存量数据。
    

## 3. 核心算术逻辑

### 3.1 加法（Addition）

基于模拟竖式加法逻辑。遍历两个序列，执行逐位求和：

$$C_i = (A_i + B_i + carry_{i-1}) \pmod{Base}$$$$carry_i = \lfloor (A_i + B_i + carry_{i-1}) / Base \rfloor$$

### 3.2 减法（Subtraction）

前提需保证 $A \ge B$。若 $A < B$，则计算 $-(B-A)$。

$$C_i = (A_i - B_i - borrow_{i-1} + Base) \pmod{Base}$$$$borrow_i = [A_i - B_i - borrow_{i-1} < 0]$$

_注意：计算结束后必须执行“前导零截断”操作。_

### 3.3 乘法（Multiplication）

- **高精度 × 单精度**：每一位直接与标量相乘，后续统一处理进位。
    
- **高精度 × 高精度**：
    
    - **普通法（O(n²)）**：基于布斯算法思想的双重循环。
        
    - **加速算法（O(n log n)）**：利用**快速傅里叶变换（FFT）**或快速数论变换（NTT）将大整数视为多项式，通过点值表示法进行卷积运算。
        

### 3.4 除法（Division）

通常采用模拟长除法（Long Division）。从最高位开始向下取余并尝试商值。对于高精度除以高精度的场景，常用二分查找结合高精度乘法进行商值的逼近。

## 4. 复杂度与性能瓶颈

|算法|时间复杂度 (n位)|空间复杂度|优化手段|
|---|---|---|---|
|**加/减法**|$O(n)$|$O(n)$|循环展开、压位|
|**基础乘法**|$O(n^2)$|$O(n)$|压位|
|**进阶乘法**|$O(n \log n)$|$O(n)$|FFT / NTT|
|**除法/取模**|$O(n^2)$|$O(n)$|Newton's Method (牛顿迭代)|

## 5. 关键边界处理（Corner Cases）

1. **零值处理**：在减法或除法产生的 `0` 需要特判，避免输出为空。
    
2. **符号位**：通常独立于数值数组，通过布尔变量标记正负。
    
3. **前导零**：在任何可能导致位数减少的操作（减法、除法）后，必须回缩容器尺寸。
